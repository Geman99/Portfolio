Product Inventory Management System (PIMS)

Production-style API built with Clean Architecture on .NET 8: versioned REST endpoints, JWT auth, RBAC, Entity Framework Core with SQL Server, Serilog logging, output caching, and Swagger / OpenAPI documentation.

Tech Used

  • .NET 8
  • Clean Architecture
  • EF Core
  • SQL Server
  • JWT
  • RBAC
  • Serilog
  • Swagger
  • API versioning

Solution architecture

  • Pims.Domain — core entities, enums, and domain models.
  • Pims.Application — contracts, DTOs, and business logic abstractions.
  • Pims.Infrastructure — EF Core DbContext, Identity integration, services, and JWT wiring.
  • Pims.Api — ASP.NET Core Web API with versioned controllers, middleware, and configuration.

API surface

  • Auth — POST /api/v1/auth/login issues JWTs; policies enforce roles on protected routes.
  • Products — get by id, search/filter/paging, create (admin), and percentage or fixed price adjustments (admin).
  • Categories — list and create (admin); many-to-many links to products.
  • Inventory — low-stock queries, create records (admin), and adjust with typed transactions (addition, subtraction, audit).

Data model

  • ASP.NET Identity tables for users and roles (e.g. Administrator, User); products with unique SKU; categories; product–category mapping; per-location inventory; transaction log for stock changes.
  • EF Core migrations for schema evolution; SQL Server (LocalDB, Express, or full instance) via configurable connection strings.

Operations & DX

  • dotnet restore, dotnet build, EF tools (dotnet ef migrations / database update), dotnet run on the API host — documented in the repo.
  • Interactive docs under /swagger; JWT configuration (issuer, audience, signing key, expiry) via appsettings.json — production keys should be strong and secret-managed.
